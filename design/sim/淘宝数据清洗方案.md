## Taobao UserBehavior 数据清洗规则

### 1. 基础过滤

* 原始字段：

```text
user_id,item_id,category_id,behavior_type,timestamp
```

* 删除字段缺失、字段数量不为 5、类型转换失败的记录。
* `user_id`、`item_id`、`category_id`、`timestamp` 必须为正整数。
* 仅保留：

```text
behavior_type == "pv"
```

* 仅保留官方时间范围：

[
2017\text{-}11\text{-}25\le t<2017\text{-}12\text{-}04
]

### 2. 去重

对以下字段完全相同的记录去重：

```text
user_id,item_id,category_id,behavior_type,timestamp
```

同一用户在不同时间重复浏览同一商品时保留。

### 3. 商品类别冲突

若同一个 `item_id` 对应多个 `category_id`：

* 使用出现次数最多的类别；
* 次数相同时选择数值较小的 `category_id`。

### 4. 低频过滤

采用非对称阈值：

[
\operatorname{count}(u)\ge 50
]

[
\operatorname{count}(i)\ge 20
]

必须迭代执行：

1. 删除 `pv` 行为数少于 50 的用户；
2. 删除总浏览次数少于 20 的商品；
3. 重新统计用户和商品频次；
4. 重复以上步骤，直到本轮没有记录被删除。

最终必须保证：

```text
每个用户至少有 50 条 pv 行为
每个商品至少被浏览 20 次
```

这里的 50 是用户总序列长度下限。后续设置 `short_len=20` 时，每个用户至少能提供约 30 条长期历史；`long_len=500` 和 `K=50` 都是最大长度，不要求每个样本填满。

### 5. 排序

最终按照以下字段稳定升序排序：

```text
user_id,timestamp,item_id
```

### 6. 本阶段不处理

* 不划分训练集、验证集和测试集；
* 不构造正负样本；
* 不进行 ID 映射；
* 不截断用户序列；
* 不提前执行 Hard Search。
