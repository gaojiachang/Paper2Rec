### 1. 脚本职责

新建：

```text
scripts/build_taobao_dataset.py
```

只做两件事：

1. 按时间划分 Train、Valid、Test；
2. 构造正样本，并为 Valid/Test 固定采样负样本。

不做 ID 映射、序列截断、Hard Search、Padding、时间分桶。训练负样本在训练时动态采样。

---

## 2. 输入

```text
data/processed/taobao-userbehavior/user_behavior_clean.parquet
```

字段：

```text
user_id,item_id,category_id,behavior_type,timestamp
```

数据已按：

```text
user_id,timestamp,item_id
```

排序。

---

## 3. 数据划分规则

使用 `Asia/Shanghai` 时区：

| 数据集     | 目标行为时间                  |
| ------- | ----------------------- |
| Warm-up | 2017-11-25，仅作为历史        |
| Train   | 2017-11-26 ～ 2017-12-01 |
| Valid   | 2017-12-02              |
| Test    | 2017-12-03              |

历史行为必须严格早于目标时间：

[
t_{\text{history}}<t_{\text{target}}
]

同一秒的其他行为不能进入历史。

---

## 4. 正样本规则

每条 `pv` 行为可作为正样本，但目标前至少有 50 条历史：

[
history_length\ge50
]

* Train：每个用户每天保留最后一个有效目标；
* Valid：每个用户当天保留最后一个有效目标；
* Test：每个用户当天保留最后一个有效目标；
* Valid/Test 目标商品必须在 Train 时间范围内出现过，否则删除并统计为冷启动商品。

---

## 5. 负样本规则

### Train

脚本不生成训练负样本，训练时动态采样。

### Valid/Test

每个正样本固定采样 99 个负样本：

```text
49 个同类别负样本
50 个全局负样本
```

负样本必须满足：

* 属于训练商品池；
* 用户在完整数据中从未浏览过；
* 不等于正样本；
* 同一候选集合内不重复。

若同类别商品不足，用全局负样本补齐。

商品采样权重：

[
P(i)\propto freq(i)^{0.75}
]

固定随机种子：

```text
seed = 2026
```

---

## 6. 输出

目录：

```text
data/processed/taobao-userbehavior/dataset/
```

文件：

```text
train_targets.parquet
valid_candidates.parquet
test_candidates.parquet
item_catalog.parquet
dataset_config.json
dataset_stats.json
```

### `train_targets.parquet`

```text
sample_id
user_id
target_item_id
target_category_id
target_timestamp
history_end_position
```

### `valid_candidates.parquet` / `test_candidates.parquet`

```text
sample_id
user_id
target_item_id
target_category_id
target_timestamp
history_end_position
negative_item_ids
```

其中：

```text
len(negative_item_ids) == 99
```

### `item_catalog.parquet`

```text
item_id
category_id
train_frequency
sampling_weight
```

---

## 7. 必要校验

脚本结束前检查：

* 所有样本历史长度不少于 50；
* Valid/Test 每个用户最多一个正样本；
* Valid/Test 每条样本恰好有 99 个负样本；
* 负样本无重复、不是正样本、未被该用户浏览；
* Valid/Test 正负商品均属于训练商品池。
